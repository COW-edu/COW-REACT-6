## 1. CSS in CSS
> “전통적인 방식 — 스타일은 별도의 CSS 파일에서 관리한다”

### 개념  
HTML, JS, React 등의 코드와는 **분리된 `.css` 파일**에서 스타일을 정의하고  
컴포넌트에 **class 이름**으로 적용하는 방식이다.

```css
/* styles.css */
.button {
  background-color: #2563eb;
  color: white;
  padding: 8px 16px;
  border-radius: 8px;
}
```
```css
// Button.jsx
import './styles.css';

function Button() {
  return <button className="button">Click</button>;
}
```

### 장점
- 브라우저 표준 방식으로 모든 환경에서 사용 가능  
- CSS 전문가나 디자이너 협업에 용이  
- CSS 전처리기(SASS, LESS 등)와 결합 용이

### 단점
- 전역 네임스페이스 충돌 위험  
- 컴포넌트 단위로 스타일 관리가 어려움  
- 동적 스타일링(state 기반) 구현이 불편

## 2. CSS in JS
> “스타일도 자바스크립트 안에서 관리한다 — 컴포넌트와 완전 결합”

### 개념  
JS(또는 TS) 코드 안에서 **JavaScript 문법으로 CSS를 작성**하는 방식이다.  
대표 라이브러리: **styled-components**, **Emotion**

```css
import styled from 'styled-components';

const Button = styled.button`
  background-color: #2563eb;
  color: white;
  padding: 8px 16px;
  border-radius: 8px;

  &:hover {
    background-color: #1e40af;
  }
`;

export default Button;
```

### 장점
- 컴포넌트 단위 캡슐화 (전역 충돌 없음)  
- props를 통한 **동적 스타일링 가능**  
- 코드 스플리팅, SSR 등과 자연스럽게 통합

### 단점
- 런타임 성능 오버헤드 발생 가능  
- 빌드 설정이 복잡 (SSR 시 스타일 누락 이슈 등)  
- 디자이너/비개발자 협업 어려움

## 3. Tailwind CSS
> “유틸리티 클래스 기반 — 미리 정의된 원자 단위 스타일 조합으로 UI 구성”

### 개념  
Tailwind는 미리 정의된 **유틸리티 클래스(Utility Class)** 를 HTML의 `class` 속성에서 직접 조합하여 사용하는 방식이다.

```css
<button className="bg-blue-600 text-white px-4 py-2 rounded-lg hover:bg-blue-800">
  Click
</button>
```

### 장점
- 별도 CSS 작성 없이 빠르게 UI 구현 가능  
- 일관된 디자인 시스템 유지 (디자인 토큰 내장)  
- 사용하지 않는 클래스 자동 제거 → 번들 크기 작음  
- 클래스 충돌 없음

### 단점
- HTML 코드가 복잡해질 수 있음 (클래스 나열)  
- 커스텀 디자인 시 설정 파일(`tailwind.config.js`) 수정 필요  
- 의미적(semantic) 구조가 약함
